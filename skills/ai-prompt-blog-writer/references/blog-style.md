# Beam307 블로그 스타일

이 문서는 `_posts/*.md` 초안을 쓸 때 참고하는 저장소 전용 스타일 가이드다.

## frontmatter 패턴

세 글 모두 아래 필드를 사용한다.

- `layout: post`
- `title`
- `author: chanhee.kim`
- `date`
- `updated_at`
- `tags`
- `image`
- `description`
- `keywords`

파일명은 보통 아래 패턴을 따른다.

```text
YYYY-MM-DD-topic-slug.md
```

특징:

- 제목은 `[Java]`, `[디자인 패턴]`처럼 카테고리성 접두어를 붙이기도 한다
- `description`은 글에서 무엇을 정리하는지 한 문장으로 설명한다
- `keywords`는 제목을 확장한 검색어 나열에 가깝다
- `updated_at`은 나중에 보강한 글에도 자주 들어간다

## 글 전개 패턴

1. 제목 다음에 바로 개념 정의나 문제의식을 제시한다
2. `왜 중요한가`, `장점/단점`, `언제 쓰는가` 같은 기본 설명을 먼저 깐다
3. 핵심 규칙, 패턴, 구현 방법을 번호 매긴 섹션으로 나눈다
4. 각 섹션마다 짧은 설명 뒤에 코드 예시나 사례를 붙인다
5. 마지막은 참고 링크나 참고 사이트로 마무리한다

중요한 점:

- 최근 글처럼 초반 결론, 비교표, 요약 테이블이 반드시 나오지는 않는다
- 별도 `결론` 섹션 없이 끝나는 경우가 많다
- 한 편의 글이 "무엇을 알아야 하는지"를 차례대로 설명하는 강의 노트형 구조에 가깝다

## 본문 구성 방식

- `##`, `###` 헤딩으로 큰 구조를 먼저 세운다
- 개념 정의 뒤에 `왜 배워야 하는가`,  `적용 기준` 같은 맥락 설명이 들어간다
- 규칙이나 안티패턴을 `1.`, `2.`, `3.` 식으로 길게 이어간다
- `나쁜 예`, `좋은 예`, `적용 전`, `적용 후` 비교를 자주 쓴다
- 하나의 주제를 여러 구현 기법으로 나눠 체계적으로 정리한다
- 코드가 설명의 중심이고, 설명 문장은 짧고 직접적이다
- 구분선 `---`를 써서 큰 섹션을 끊는다
- 먼저 정의와 필요성을 설명하고
- 그 다음 규칙 또는 기법을 번호로 나누고
- 각 번호마다 예시를 붙이는 방식이 가장 자연스럽다

## 문체와 언어

- 한국어 기술 설명문 스타일이다
- 최근 글보다 덜 대화체이고, 더 강의 노트에 가깝다
- 실무 감각은 있지만 의견보다 설명과 정리가 앞선다
- 영어 용어는 괄호로 병기하거나 그대로 유지한다
- 문장은 짧고 단정적으로 쓴다

자주 쓰이는 표현:

- `...이란?`
- `왜 ...를 배워야 할까?`
- `장점`
- `단점`
- `나쁜 예`
- `좋은 예`
- `참고`

## Markdown 관례

- 큰 제목은 `##`, 세부 항목은 `###` 또는 `####`를 사용한다
- 번호 매긴 목록과 불릿 목록을 많이 쓴다
- 코드 블록은 설명 직후 바로 붙는다
- 강조가 필요한 용어는 `**굵게**` 처리한다
- 필요하면 `---`로 큰 흐름을 끊는다

## 이 블로그답게 보이게 만드는 요소

- 개념을 바로 정의한다
- 독자가 왜 이 주제를 배워야 하는지 설명한다
- 규칙이나 패턴을 번호로 나눠 차례대로 설명한다
- 말로만 끝내지 않고 코드 예시를 붙인다
- 글 마지막에 참고 링크를 남긴다
